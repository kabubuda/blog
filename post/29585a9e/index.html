利用AdGuardHome搭建无污染DNS屏蔽广告

安装AdGuardHome

AdGuardHome 作为开源软件 可以在AdGuardHome官方Github地址 找到官方说明文档

Linux本地安装

Run the following command in your terminal:

或 下载后

进入安装目录 运行安装命令

  • 对于Fedora Linux 及其分支: 安装目录为 /usr/local/bin

选项说明:

  • -c <channel> 使用特定通道.
  • -r 安装 AdGuard Home;
  • -u 卸载 AdGuard Home;
  • -v 详细信息输出;

作为服务启动：

  • AdGuardHome -s uninstall:卸载服务.
  • AdGuardHome -s start: 启动服务.
  • AdGuardHome -s stop: 停止服务
  • AdGuardHome -s restart: 重启服务
  • AdGuardHome -s status: 查看服务状态

Docker安装

官方Docker安装说明

可选参数

  • -p 53:53/tcp -p 53:53/udp: DNS 端口。即其他设备访问 AdGuard Home 进行 DNS 解析的默认端口。因为部分系统不支持自定义 DNS 端口，所以不建议自定义。部署前务必要查看是否有其它程序占用。
  • -p 67:67/udp -p 68:68/tcp -p 68:68/udp: DHCP 端口。除非想代替你路由上的 DHCP 服务器，否则用不到
  • -p 80:80/tcp -p 443:443/tcp -p 443:443/udp -p 3000:3000/tcp: 管理面板，以及HTTPS/DNS-over-HTTPS服务器
  • -p 853:853/tcp: DNS-over-TLS 服务。不使用相关功能可忽略。
  • -p 784:784/udp -p 853:853/udp -p 8853:8853/udp: DNS-over-QUIC 服务 只需保留一个或两个
  • -p 5443:5443/tcp -p 5443:5443/udp: DNSCrypt 服务

开放端口

  • 3000/TCP 初始化面板
  • 80/TCP web interface;
  • 53/UDP DNS 服务

其他端口见上 如果docker映射了其他端口则开放相应端口

添加规则

最后 也是最重要的一步 到 过滤器 > DNS拦截列表 中添加一些拦截规则，可以帮助分辨哪些请求内容是广告，下面是几个我使用的规则

名称 地址 简介
ADRules https://github.com/Cats-Team/AdRules 基于中文区的广告规则
anti-AD https://anti-ad.net/easylist.txt 致力于成为中文区命中率最高的广告过滤列表
ADgk https://raw.githubusercontent.com/banbendalao/ADgk/master/ADgk.txt 针对手机软件的广告
EasyList-China https://easylist-downloads.adblockplus.org/easylistchina+easylist.txt Adblock Plus官方规则
