红黑树概述

发表于2022-07-11|更新于2022-08-29|编程
|阅读量:

概述

红黑树的由来

在使用二叉搜索树时，我们知道它的问题是容易退化成一条链。

这样它的查找时间复杂度从O(log N)退化成O(N)

为了防止这种情况，引入了平衡二叉树(AVL)，它通过旋转操作保持左右子树高度的平衡

但是，AVL也有一些问题：

  • AVL的左右子树高度差不能超过1，每次进行插入/删除操作时，几乎都需要通过旋转操作保持平衡
  • 在频繁进行插入/删除的场景中，频繁的旋转操作使得AVL的性能大打折扣

为了保持插入和查找性能的平衡，红黑就被发明出来了

  • 红黑树通过牺牲严格的平衡，换取插入/删除时少量的旋转操作，整体性能优于AVL
  • 红黑树插入时的不平衡，不超过两次旋转就可以解决；删除时的不平衡，不超过三次旋转就能解决
    红黑树的红黑规则，保证最坏的情况下，也能在O( log N ) 时间内完成查找操作。

所以，红黑树是一种特化的AVL树，它是一棵自平衡树（不是绝对的平衡）的二叉查找树（BST）

红黑树的规则

一棵经典的红黑树：

红黑树性质：

  • 节点不是黑色，就是红色（非黑即红）
  • 根节点为黑色
  • 叶节点为黑色（叶节点是指末梢的空节点 Nil或Null）
  • 一个节点为红色，则其两个子节点必须是黑色的（根到叶子的所有路径，不可能存在两个连续的红色节点）
  • 每个节点到叶子节点的所有路径，都包含相同数目的黑色节点（相同的黑色高度）

说明：

  • 约束4和5，保证了红黑树的大致平衡：根到叶子的所有路径中，最长路径不会超过最短路径的2倍。

  • 使得红黑树在最坏的情况下，也能有O (logN) 的查找效率关于叶子节点：Java实现中，null代表空节点，无法看到黑色的空节点，反而能看到传统的红色叶子节点默认新插入的节点为红色：因为父节点为黑色的概率较大，插入新节点为红色，可以避免颜色冲突

2-3-4树

2-3-4树简介

2-3-4树是四阶B树（Balance Tree）。它属于一种多路查找树，它的结构有以下限制：

所有的叶子节点拥有相同的深度

节点只能是 2-节点， 3-节点，4节点之一； 超过4个结点就会裂变

  • 2-节点： 包含1个元素的节点，有2个子节点；
  • 3-节点： 包含2个元素的节点，有3个子节点
  • 4-节点： 包含3个元素的节点，有4个子节点

裂变过程：

每当4结点超过超过三个元素就进行裂变，将多出的元素上移到父节点，如果父节点元素超过3个则继续裂变，以此类推
